Section 04
BWallSelf Decode Engine — Formation Prediction
How This Works
Enter a solar event's parameters. The engine computes the UBP solar message weight (flare class + AR complexity + Kp contribution), runs a simulated successive-cancellation decode against the Golay [24,12,8] codebook, and outputs a predicted geometry class, site preference, and formation window. Confidence is low until the methodology is formally validated — predictions should be logged and compared against subsequent formation reports.

Critical Gap — OCTAD_TO_GEOMETRY_MAP
The solar→Golay weight→geometry lookup table is the primary missing component of this framework. The current decode maps solar message weight to the nearest weight-8 or weight-12 octad class, then to a D₂/C₄ᵥ/D₆ geometry via heuristic. This must be replaced with a formally derived bijection before predictions carry scientific weight. Track this gap in the database notes field.
